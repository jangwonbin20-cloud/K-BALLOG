# K-BALLOG 프로젝트 청사진

## 1. 프로젝트 개요

K-BALLOG는 대한민국 아마추어 축구 리그의 소식을 통합 제공하는 웹 플랫폼입니다. 사용자는 지역별(서울, 수원, 인천, 용인 등) 리그의 경기 일정, 결과, 순위, 관련 뉴스 등 다양한 정보를 손쉽게 확인하고, 자신이 응원하는 팀과 선수들의 소식을 접할 수 있습니다.

## 2. 디자인 시스템 및 레이아웃

### 2.1. 시각적 정체성

- **테마**: 전체적으로 어두운 테마(Dark Mode)를 적용하여 모던하고 집중도 높은 사용자 경험을 제공합니다.
- **컬러 팔레트**: 주조색으로 `--dark-bg` (`#1a1c23`), 카드 배경색으로 `--card-bg` (`#24262e`), 그리고 강조색으로 강렬한 빨간색 계열의 `--primary-accent` (`#e4002b`)를 사용합니다.
- **아이콘**: Font Awesome 아이콘을 활용하여 메뉴와 버튼의 시각적 이해를 돕습니다.
- **반응형 디자인**: 모바일, 태블릿, 데스크톱 등 다양한 화면 크기에 대응하는 반응형 레이아웃을 갖추고 있습니다.

### 2.2. 핵심 컴포넌트: 통합 카드 (`.unified-card`)

뉴스, 리그 바로가기, 팀 목록 등 다양한 콘텐츠를 일관된 형태로 표시하기 위한 재사용 가능한 카드 컴포넌트입니다. 이미지, 제목, 설명 텍스트로 구성되며, 마우스를 올리면 부드러운 애니메이션 효과가 적용되어 사용자 인터랙션을 유도합니다.

### 2.3. 페이지 레이아웃

- **메인 콘텐츠 + 사이드바 레이아웃**: 주요 정보를 담는 넓은 메인 콘텐츠 영역과 추가 정보(바로가기 링크, 매치 센터 등)를 제공하는 좁은 사이드바 영역으로 구성된 2단 레이아웃을 사용합니다.
- **그리드 레이아웃**: 여러 개의 통합 카드를 격자 형태로 배열하여 보여주는 데 사용됩니다. 화면 크기에 따라 열의 개수가 자동으로 조절됩니다.

## 3. 개발 이력 및 변경 사항

### 3.1. 초기 디자인 개편 (1차)

- **목표**: 분산되어 있던 콘텐츠 표시 방식을 통일하여 사용자 경험의 일관성을 확보합니다.
- **변경 사항**:
    1.  **통합 카드 도입**: `.unified-card` 스타일을 `style.css`에 정의하여 사이트 전반의 카드 디자인을 통일했습니다.
    2.  **홈페이지(`index.html`) 구조 변경**: 여러 섹션을 '지역별 리그 바로가기'로 통합하고, 통합 카드를 사용한 그리드 레이아웃으로 개편했습니다.
    3.  **지역 페이지(`seoul.html`) 리팩토링**: 하위 리그 링크를 통합 카드 디자인으로 교체하여 통일성을 강화했습니다.

### 3.2. K7 리그 페이지 디자인 고도화 (2차)

- **목표**: K7 리그 관련 페이지의 정보 전달력을 높이고, 시각적으로 더욱 매력적인 페이지를 구성합니다.
- **변경 사항**:
    1.  **`seoul-k7.html` 디자인 적용**.
    2.  **`seoul-k7-seongbuk.html` 디자인 통일**.

### 3.3. 내비게이션 확장 및 신규 페이지 생성 (3차)

- **목표**: 뉴스 및 커뮤니티 기능을 위한 기반을 마련하고, 모든 페이지에서 일관된 내비게이션을 제공합니다.
- **변경 사항**:
    1. **내비게이션 메뉴 추가**: 모든 주요 페이지의 헤더에 '뉴스'와 '커뮤니티' 메뉴 링크를 추가했습니다.
    2. **`news.html` 및 `community.html` 페이지 생성**: 뉴스 및 커뮤니티 페이지의 기본 구조를 마련했습니다.
    3. **콘텐츠 및 스타일 추가**: 신규 페이지에 초기 샘플 콘텐츠를 추가하고, `style.css`에 관련 스타일을 정의했습니다.

### 3.4. 홈페이지 다국어 지원 기능 추가 (4차)

- **목표**: 홈페이지(`index.html`)에 다국어 지원 기능을 추가하고, 사용자가 쉽게 접근할 수 있도록 UI를 개선합니다.
- **변경 사항**:
    1. **언어 선택 UI 추가 및 재배치**: 헤더의 우측 끝에 언어 선택 드롭다운 메뉴를 배치하여 접근성을 높입니다. (`.header-actions` 영역으로 이동)
    2. **콘텐츠 번역**: 홈페이지의 주요 텍스트에 대한 번역 데이터를 `main.js`에 준비합니다.
    3. **번역 스크립트 작성**: 사용자가 언어를 선택했을 때, 페이지의 텍스트를 동적으로 변경하는 JavaScript 코드를 `main.js`에 추가합니다.
    4. **스타일 적용 및 수정**: `style.css`에서 언어 선택 드롭다운의 스타일을 수정하고, 새로운 위치에 맞게 헤더 레이아웃을 조정합니다.

### 3.5. 커뮤니티 새 글 작성 기능 활성화 (5차)

- **목표**: 사용자가 커뮤니티 게시판에 새로운 글을 작성하고, 작성된 글 목록을 볼 수 있도록 기능을 활성화합니다.
- **변경 사항**:
    1.  **`create-post.html` 생성**: 제목과 내용 입력 폼을 포함한 새 글 작성 페이지를 추가했습니다.
    2.  **`community.html` 수정**: '새 글 작성' 버튼을 `create-post.html`로 연결하고, 기존의 정적 게시물 목록을 삭제하여 동적 로딩을 준비했습니다.
    3.  **`main.js` 기능 추가**:
        -   새 글 폼 제출 시, 입력된 데이터를 `localStorage`에 저장하는 로직을 구현했습니다.
        -   `community.html` 로드 시, `localStorage`에서 게시물 목록을 가져와 동적으로 화면에 표시하는 기능을 추가했습니다. 게시물이 없을 경우 안내 메시지를 표시합니다.
    4.  **`style.css` 스타일 추가**:
        -   `create-post.html`의 폼과 버튼에 대한 스타일을 추가했습니다.
        -   게시물이 없을 때 표시되는 `.no-posts-message`의 스타일을 추가하여 디자인 통일성을 유지했습니다.

### 3.6. 홈페이지 인증 기능 추가 (6차)

- **목표**: 사용자가 이메일과 비밀번호를 사용하여 회원가입하고 로그인/로그아웃할 수 있는 기능을 홈페이지에 추가합니다.
- **변경 사항**:
    1. **`index.html` 구조 변경**:
        - 헤더에 '로그인', '회원가입' 버튼 및 로그인 시 사용자 정보를 표시할 영역을 추가했습니다.
        - 로그인 및 회원가입을 위한 별도의 모달 창 UI를 페이지에 포함시켰습니다.
    2. **`style.css` 스타일 추가**:
        - 헤더의 인증 관련 버튼(`btn-auth`), 사용자 정보(`user-info`) 스타일을 추가했습니다.
        - 화면 전체를 덮는 모달 오버레이(`.modal-overlay`)와 모달 콘텐츠(`.modal-content`)에 대한 스타일을 정의하여 디자인 일관성을 확보했습니다.
    3. **`main.js` 기능 구현**:
        - **모달 제어**: 로그인 및 회원가입 버튼 클릭 시 해당하는 모달 창을 열고, 닫기 버튼이나 모달 바깥 영역 클릭 시 창을 닫는 기능을 구현했습니다.
        - **사용자 데이터 관리**: 회원가입 시 사용자 정보를 `localStorage`에 배열 형태로 저장하고, 로그인 시 이 정보를 확인하여 인증을 처리합니다.
        - **UI 동적 업데이트**: 사용자의 로그인 상태에 따라 헤더의 버튼(로그인/회원가입 또는 로그아웃/사용자 이름)을 동적으로 변경하는 `updateAuthState` 함수를 작성했습니다.

### 3.7. '이적' 섹션 추가 (7차)

- **목표**: 사이트 전반에 '이적' 관련 기능을 통합하여 사용자에게 선수 이적 시장에 대한 정보를 제공합니다.
- **변경 사항**:
    1.  **`transfer.html` 페이지 생성**: '이적 시장'이라는 제목과 함께 이적 관련 콘텐츠가 표시될 기본 구조를 갖춘 신규 페이지를 추가했습니다.
    2.  **전체 내비게이션 업데이트**: `index.html`, `match.html`, `search.html`, `news.html`, `community.html`, `contact.html`, `create-post.html` 등 모든 주요 페이지의 헤더 내비게이션 바에 '이적' 메뉴를 추가하여 일관된 사용자 경험을 제공합니다.

### 3.8. 지역 페이지 헤더 업데이트 (8차)

- **목표**: 서울, 수원, 인천, 용인 각 지역 페이지의 헤더를 최신 디자인으로 통일하고 '이적' 메뉴를 추가합니다.
- **변경 사항**:
    1. **`seoul.html`, `suwon.html`, `incheon.html`, `yongin.html` 페이지 헤더 수정**:
        - 각 지역 페이지의 헤더 내비게이션 바에 '이적' 링크를 추가했습니다.
        - 지역 선택 드롭다운 메뉴에 기본 선택 옵션을 추가하고, 각 페이지에 해당하는 지역이 기본으로 선택되도록 수정했습니다.

### 3.9. SSSL 정보 업데이트 (9차)

- **목표**: SSSL(서울 주말 축구 리그)이 외국인들을 위한 리그라는 정보를 명확하게 전달합니다.
- **변경 사항**:
    1. **`seoul-sssl.html` 콘텐츠 수정**: 페이지 제목, 메인 헤딩, 설명에 '외국인 리그'임을 명시하고, 글로벌한 느낌을 강조하기 위해 아이콘을 변경했습니다.
    2. **`seoul.html`의 SSSL 카드 정보 수정**: 서울 지역 페이지에서 SSSL을 '외국인 리그' 및 '국제 교류 리그'로 설명하는 문구를 업데이트했습니다.

### 3.10. FC CREO 팀 페이지 사진 업데이트 (10차)

- **목표**: FC CREO 팀 페이지에 실제 팀 단체 사진을 추가하여 페이지의 완성도를 높입니다.
- **변경 사항**:
    1. **`team-creo.html` 사진 교체**: 기존의 플레이스홀더 이미지를 전달받은 실제 팀 단체 사진으로 교체하고, 사진 설명을 "FC CREO 창단 1주년 단체 사진"으로 수정했습니다.
    2. **`team-creo.html` 코드 정리**: 불필요한 헤더 코드를 제거하여 코드를 정리했습니다.

### 3.11. FC CREO 팀 페이지 사진 변경 (11차)

- **목표**: FC CREO 팀 페이지의 단체 사진을 사용자가 새로 요청한 이미지로 변경합니다.
- **변경 사항**:
    1. **`team-creo.html` 사진 교체**: 기존 팀 단체 사진을 사용자가 새로 전달한 이미지로 교체하고, 사진 설명을 "FC CREO 팀 사진"으로 수정했습니다.

### 3.12. FC CREO 팀 페이지 사진 오류 수정 (12차)

- **목표**: FC CREO 팀 페이지의 이미지가 깨져 보이는 오류를 수정합니다.
- **변경 사항**:
    1. **`team-creo.html` 이미지 경로 수정**: 잘못된 이미지 경로를 올바른 경로로 수정하여 이미지가 정상적으로 표시되도록 했습니다.
